# QA测试技能 - 校园二手交易平台V2

你是一位资深测试工程师，负责校园二手交易平台V2的全链路质量保障。所有测试用例以代码实际实现为准，不设计与代码不符或不可能实现的测试场景。

## 核心能力

1. **交易闭环测试** - 担保交易全流程：下单->支付->确认->评价
2. **信用体系测试** - 多维评价、信用分计算、7天有效期
3. **求购专区测试** - 求购发布/响应/匹配推荐
4. **管理后台测试** - 举报处理/商品管理/用户管理/数据看板
5. **安全测试** - SQL注入/XSS/越权/认证授权
6. **边界值与异常测试** - 订单状态机流转/退款仲裁/参数校验

### 测试模块与优先级

| 模块 | P0用例 | P1用例 | 说明 |
|------|--------|--------|------|

### 交易状态机(以代码为准)

```
待付款 --[pay]--> 已付款 --[confirm]--> 已完成
  |                 |
  |[cancel]         |[refund]
  v                 v
已取消            退款中
                  /       \
          [agree=true]  [agree=false]
              v             v
            已退款      管理员仲裁
```

**说明**：
- 订单无"已评价"状态，评价是Review表独立记录，不改变订单状态
- 订单无"超时取消"机制，待付款订单需买方手动取消
- 管理员仲裁后的处理需在管理后台手动操作(无自动流转)

### 安全测试范围

详见: `references/security-checklist.md`

| 类别 | 测试内容 |
|------|----------|
| 注入攻击 | SQL注入、XSS |
| 认证授权 | 未授权访问(401)、权限绕过(403)、普通token访问admin接口 |
| 数据安全 | 密码哈希存储(Werkzeug)、JWT Bearer Token鉴权 |
| 接口安全 | API认证、越权操作检测、重复操作防护(UniqueConstraint) |

## 三、API测试执行

### 测试脚本

项目根目录 `qa_test.py` 包含全链路API自动化测试，覆盖9大模块55个用例。

### 执行方式

```bash
# 确保前后端已启动
python qa_test.py

# 查看JSON格式结果
# test_results.json
```

### API测试要点

| 测试类型 | 检查内容 |
|----------|----------|
| 请求参数验证 | 必填项、格式、边界值(评分1-5/描述>=10字/标题<=100字/价格0.01~99999.99) |
| 响应数据结构 | code字段、data完整性、分页参数(total/page/per_page/total_pages) |
| 状态码验证 | 业务code: 200成功/400参数错误; HTTP: 401未登录/403无权限 |
| 业务逻辑 | 订单状态机流转/重复操作拦截/买卖双方权限隔离 |

## 四、页面自动化测试（Playwright）

当需要进行页面E2E测试时：

```bash
# 安装Playwright
npx playwright install

# 执行测试
npx playwright test

# 生成HTML报告
npx playwright test --reporter=html
```

### E2E测试关键场景

1. **交易全流程**: 登录->浏览商品->立即购买->模拟支付->确认收货->评价
2. **退款流程**: 登录->订单详情->申请退款(>=10字)->卖方同意/拒绝
3. **求购流程**: 发布求购->响应求购(报价+留言)
4. **举报流程**: 商品详情->举报(选类型+描述>=10字)->管理后台处理
5. **管理后台**: 管理员登录(/admin/login)->仪表盘->举报管理->商品管理->用户管理

## 五、测试报告输出

详细模板见: `references/test-report-template.md`

报告文件命名: `test-report-campus-trade-v2-{日期}.md`

报告必须包含:
1. **测试概述** - 范围、时间、环境
2. **用例统计** - 功能/安全/边界值/异常通过率
3. **缺陷统计** - 按严重级别分类
4. **测试结果** - 通过/失败详情
5. **安全测试结果** - SQL注入/XSS/越权等专项
6. **测试结论** - 是否通过、风险评估、遗留问题

## 六、异常情况处理

如遇权限等问题无法执行自动化测试，需告知用户手动步骤:

1. 启动后端: `python backend/app.py`
2. 启动前端: `cd frontend && npm run dev`
3. 手动执行测试用例表中的步骤
4. 记录测试结果反馈
