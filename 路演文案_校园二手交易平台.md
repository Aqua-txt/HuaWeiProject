# 校园二手交易平台（拾光集市）路演文案

---

## 1. 一页摘要

- **项目名称：** 拾光集市——校园二手交易平台
- **一句话概述：** 基于担保交易+信用评价+双向匹配的校园二手交易服务平台，让每一笔校园交易都有保障、有记录、有信用
- **预期交付时间：** 已完成全功能开发，当前可演示
- **请求决策事项：** 确认下一阶段接入真实微信支付、推广上线计划

---

## 2. 背景与机会

### 当前状态

高校校园内二手交易需求旺盛，但现有渠道存在严重缺陷：

| 渠道 | 问题 |
|------|------|
| 微信群/QQ群 | 信息分散、无法搜索、刷屏快、无法留存 |
| 闲鱼/转转 | 非校园场景、缺乏校友信任背书、面交不便 |
| 校内贴吧/论坛 | 无支付保障、无评价体系、交互体验差 |
| 校内自建平台 | 多数停留在信息发布，缺乏交易闭环和管理能力 |

### 核心机会

> **校园二手交易是一个"高频刚需低信任"的市场——需求真实存在，但信任缺失导致交易效率极低。**

- **数据支撑：** 每位大学生年均闲置物品价值约 800-1500 元，全市场规模超百亿
- **痛点量化：** 校内微信群中"求购"消息占比超 30%，但匹配效率不足 10%（信息刷屏快、无法搜索）
- **不做的代价：** 竞品（闲鱼校园版等）正在渗透，窗口期有限

### 五大核心痛点

```
1. 找不到 → 信息分散，无法统一搜索
2. 不敢买 → 线下转账无保障，货不对板无法追责
3. 不信任 → 每次交易都是"一次性博弈"，好卖家无法积累信誉
4. 没人卖 → 想买的冷门物品没人发布，需求无法表达
5. 没人管 → 违规内容靠用户反馈，平台缺乏治理能力
```

## 3. 方案概览

### 产品定位

**从"信息撮合工具"升级为"交易服务平台"**

| 维度 | 传统校内平台 | 拾光集市 |
|------|------------|---------|
| 信息 | 群内刷屏，无法搜索 | 结构化发布 + 关键词搜索 + 分类筛选 |
| 交易 | 线下转账，风险自担 | 担保交易 + 订单闭环 + 退款机制 |
| 信任 | 口耳相传，无积累 | 三维评价 + 信用分 + 历史记录 |
| 匹配 | 只能买家找卖家 | 求购专区 + 智能推荐，双向匹配 |
| 治理 | 无人管理 | 管理后台 + 数据看板 + 举报处理 |

### 核心体验流程

```
┌─────────────────────────────────────────────────────────┐
│                    买方视角                              │
│                                                         │
│  搜索/浏览商品 → 查看详情+卖家信用 → 收藏/联系卖家        │
│       ↓                                                 │
│  立即购买 → 担保支付(平台托管) → 面交 → 确认收货          │
│       ↓                                                 │
│  平台打款卖方 → 互评三维度 → 信用分积累                   │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    卖方视角                              │
│                                                         │
│  发布商品(拍照+描述+定价) → 等待买家咨询                  │
│       ↓                                                 │
│  收到订单 → 确认收款(货款到账) → 互评 → 信用积累          │
│       ↓                                                 │
│  或：浏览求购专区 → 响应求购报价 → 达成交易               │
└─────────────────────────────────────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│                    管理员视角                            │
│                                                         │
│  登录管理后台 → 数据看板(平台健康度一览)                   │
│       ↓                                                 │
│  举报管理(下架/封禁) → 商品管理 → 用户管理                │
└─────────────────────────────────────────────────────────┘
```

### 技术架构

```
┌──────────────────────────────────────────────┐
│           Vue3 + Element Plus SPA             │
│         (用户端 + 管理端 双路由)               │
└──────────────────┬───────────────────────────┘
                   │ REST API (轮询聊天)
┌──────────────────▼───────────────────────────┐
│           Flask + SQLAlchemy API              │
│    (9个路由蓝图 / 49个API端点 / JWT双通道)     │
└──────────────────┬───────────────────────────┘
         ┌─────────┼─────────┐
         ▼         ▼         ▼
    ┌────────┐ ┌────────┐ ┌──────────┐
    │ SQLite │ │ uploads│ │ Mock Pay │
    │ 12张表 │ │ 图片存储│ │ 模拟支付  │
    └────────┘ └────────┘ └──────────┘
```

### 为什么选这个方案

| 方案 | 优势 | 劣势 | 结论 |
|------|------|------|------|
| **全栈自建（当前方案）** | 完全可控、定制灵活、数据自有 | 开发量较大 | ✅ 校园场景定制需求强，自建最优 |
| 基于闲鱼开放平台 | 流量现成 | 功能受限、无法定制交易流程 | ❌ 无法实现担保交易和信用体系 |
| 微信小程序 | 触达便捷 | 审核周期长、功能受限 | 🔜 V4.0 可作为触达渠道补充 |

## 4. 功能全景展示

### 4.1 首页——商品发现

> **核心价值：让找东西像刷淘宝一样简单**

- 搜索栏：关键词实时搜索（标题+描述模糊匹配）
- 分类筛选：10大分类一键切换
- 排序选择：最新发布 / 价格升序 / 价格降序
- 瀑布流卡片：图片 + 价格 + 标题 + 成色 + 卖家头像 + 发布时间
- 模态详情面板：点击卡片弹出详情，无需跳转页面

**演示要点：** 搜索"高数"→ 筛选"教材"分类 → 按价格排序 → 点击查看详情

### 4.2 商品详情——信息+信任+行动

> **核心价值：一次看清商品全貌和卖家信用，降低决策成本**

- 图片轮播：触摸滑动 + 计数器 + 收藏角标
- 卖家信息栏：头像 + 昵称 + 认证校友标签 + **信用分徽章**
- 商品信息：标题 + 价格 + 分类标签 + 成色标签 + 描述
- 底部操作栏：**收藏** / **举报** / **联系卖家** / **立即购买**

**演示要点：** 查看信用分 → 点击收藏 → 点击"联系卖家"进入聊天

### 4.3 发布商品——拍照即卖

> **核心价值：3分钟完成发布，降低卖方门槛**

- 表单：标题 + 描述 + 价格 + 分类(10种) + 成色(4种)
- 图片上传：最多6张，拖拽/点击上传，实时预览
- 编辑模式：支持修改已发布商品的所有字段

**演示要点：** 上传图片 → 填写信息 → 发布成功 → 在首页看到

### 4.4 担保交易——像闲鱼一样安全

> **核心价值：平台担保支付，消除"先转账还是先发货"的信任困境**

```
买家下单 → 商品自动下架 → 确认支付(平台托管)
    → 双方面交 → 买家确认收货 → 平台打款卖家
```

- 订单状态全生命周期：待付款 → 已付款 → 已完成 → 已评价
- 订单列表：我买到的 / 我卖出的 Tab 切换
- 订单详情：状态时间线 + 商品信息 + 双方信息 + 操作按钮

**演示要点：** 点击"立即购买" → 确认支付 → 查看订单 → 确认收货

### 4.5 退款机制——交易有保障

> **核心价值：不满意可退款，平台提供纠纷处理通道**

- 买家申请退款：填写原因（≥10字）
- 卖家响应：同意退款（货款退回，商品重新上架）/ 拒绝退款（进入管理员仲裁）
- 订单状态：已付款 → 退款中 → 已退款 / 管理员仲裁

**演示要点：** 申请退款 → 卖家同意 → 查看退款状态

### 4.6 信用评价——好卖家越卖越好

> **核心价值：让诚信可量化、可积累，好卖家脱颖而出**

- 三维度评分：沟通态度 / 描述相符 / 交易速度（各1-5星）
- 信用分公式：**沟通×0.2 + 描述×0.5 + 速度×0.3**
- 信用展示：商品详情页卖家信用徽章 + 个人中心信用分
- 评价规则：每订单仅评一次，7天有效期，防刷评

**演示要点：** 交易完成后进入评价页 → 三维度打分 → 查看信用分更新

### 4.7 求购专区——不再只有卖家市场

> **核心价值：让需求主动找人，从"买家找卖家"到"双向匹配"**

- 发布求购：标题 + 描述 + 预算范围 + 分类 + 期望成色
- 云标签式列表：一目了然看到预算和分类
- 响应求购：卖方报价 + 留言 + 可关联自己商品
- **智能推荐**：求购详情页自动推荐同类目+价格在预算范围内的上架商品（最多5个）

**演示要点：** 发布"求购一本高数教材" → 查看智能推荐 → 卖方响应报价

### 4.8 私信聊天——交易沟通零距离

> **核心价值：基于商品的即时沟通，交易细节不用加微信**

- 会话列表：对方头像 + 未读角标 + 最后消息 + 关联商品缩略图
- 聊天界面：消息气泡区分双方 + 实时轮询(3s) + 自动标记已读
- 导航栏角标：未读消息总数实时显示
- 会话复用：同一买家+商品+卖家不重复创建会话

**演示要点：** 从商品详情"联系卖家" → 发送消息 → 查看会话列表角标

### 4.9 管理后台——平台可管可控

> **核心价值：数据驱动的运营管理，让平台治理有抓手**

- **数据看板**：
  - 概览卡片：今日订单/发布/新用户，商品/用户/订单总数
  - 趋势图表：近30天订单/发布/新用户走势
  - 分类分布：商品分类占比统计
- **举报管理**：待处理/全部筛选 → 驳回/下架/下架+封禁
- **商品管理**：搜索+筛选 → 强制上下架
- **用户管理**：搜索(学号/昵称) → 查看信用分 → 封禁/解封

**演示要点：** 登录管理后台 → 查看数据看板 → 处理一条举报 → 封禁一个用户

## 5. 核心竞争力

### 与竞品对比

| 能力 | 微信群 | 闲鱼 | 校内自建平台 | **拾光集市** |
|------|--------|------|------------|------------|
| 结构化搜索 | ❌ | ✅ | 部分 | ✅ |
| 担保交易 | ❌ | ✅ | ❌ | ✅ |
| 三维信用评价 | ❌ | ❌ | ❌ | ✅ |
| 求购专区 | 低效发消息 | ✅ | ❌ | ✅+智能推荐 |
| 管理后台 | ❌ | ✅ | ❌ | ✅ |
| 校园社交信任 | ✅ | ❌ | ✅ | ✅ |
| 校友认证标识 | ❌ | ❌ | ❌ | ✅ |

**差异化核心：唯一同时具备"担保交易 + 三维信用 + 求购匹配 + 运营管理"的校园专属平台**

### 技术亮点

- **JWT 双通道认证**：用户端与管理端独立 token，安全隔离
- **增量消息轮询**：after_id 机制，仅拉取新消息，减少传输量
- **智能求购推荐**：同类目 + 预算范围匹配，无需 AI 即可实现有效推荐
- **图片自动压缩**：Pillow 压缩至 1200×1200，节省存储和带宽
- **订单状态机**：完整的订单生命周期管理，退款+仲裁闭环

## 8. 风险与备选

| 风险 | 概率 | 影响 | 应对策略 |
|------|------|------|----------|
| 真实微信支付审核不通过 | 中 | 高 | V3.0 用模拟支付先行上线，V3.1 切换真实支付 |
| 用户习惯微信群不愿迁移 | 高 | 高 | 冷启动策略：先导入高价值商品（毕业季教材），制造"这里有好货"的认知 |
| 交易纠纷处理人力不足 | 中 | 中 | V3.0 管理员仲裁兜底，V3.1 引入自动规则（超时自动确认/退款） |
| SQLite 并发性能瓶颈 | 低 | 中 | 校内场景并发量有限（≤200 QPS），SQLite 足够；远期可迁移 PostgreSQL |
| 图片存储空间不足 | 低 | 低 | Pillow 压缩至 1200×1200，单图约 100-200KB，1万商品约 1-2GB |
| 刷评/刷单风险 | 低 | 中 | 每订单仅评一次+7天有效期+信用分加权算法防极端值 |

### 需要决策的事项

- **支付方案**：V3.1 接入微信支付（需商户号）还是支付宝（门槛更低）？→ 建议微信支付（校园场景微信渗透率>90%）
- **推广策略**：先做一个学校还是同时铺开？→ 建议先单校验证，跑通后再复制
- **移动端**：微信小程序 vs 原生App？→ 建议小程序（开发成本低+微信生态触达）

## 10. 总结

**拾光集市**不是又一个信息发布平台——

它是**唯一一个**为校园场景量身打造、同时具备以下五大能力的交易服务平台：

```
🔍 结构化搜索 —— 找到货
💰 担保交易   —— 敢交易
⭐ 三维信用   —— 能信任
📢 求购专区   —— 有人卖
🛡️ 管理后台   —— 可治理
```

**从"信息撮合"到"交易服务"，让校园交易像当面交易一样安全，像线上平台一样高效。**
